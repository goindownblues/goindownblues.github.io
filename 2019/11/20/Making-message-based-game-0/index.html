Making message based game #0

Nov 20, 2019

0. 불씨를 살리다.

올해 초 해보려다 접었던 유튜브 컨텐츠를 위해 개발한 메신저 형태의 컨텐츠 개발툴이 있었다. 애플의 Message나 여느 메신저의 형태를 갖고 있으며, 화면을 탭하거나 클릭하면 다음 메시지가 등장해 메시지를 주고 받는 모습을 가상으로 보여줄 수 있었다. 나는 이 툴을 영상 캡쳐를 통해 유튜브 컨텐츠를 개발 해보려고 했었다.

하지만 정작 중요한 소재를 준비하는 과정에서는 부족한 부분이 너무나도 많았고, 잠시 외장 SSD에 쳐박아 두게 된 프로젝트 중 하나가 되었다. 문득 이 프로젝트가 생각난 것은 그제 저녁 이었다.

1. 꺼진 불도 다시 보자.

우리 선조님들은 많은 말씀들을 남기셨고, 꺼진 불과 접힌 프로젝트도 다시 보자는 명언 또한 남기셨다. (??) 다시 꺼내어 본 프로젝트는 참 가관 이었는데, 아무리 기준을 낮게 보아도 엉망인 구조 탓 이었다.
일단 버튼을 추가해서 인터렉션을 가할 수 있다면 좋겠다는 생각에 버튼 기능만 간단히 추가해보고, json 형태의 db 파일에 간단한 내용을 추가하여 커뮤니티에서 공유해 보았다.

아직은 진척도도 많지 않고, 목표도 명확하지 않은 프로젝트라 많은 피드백은 없었지만, 퍼포먼스에 대한 피드백은 있었기 때문에 이참에 조금 더 힘을 써보기로 했다.

기존 버전에서는 jQuery로 작업했었다면, 지난 토이 프로젝트 작업 당시에 배운 React로 작업 해보기로 결정한 것이다. React로 새로 판을 깔기 시작한 것이 어제 늦은 저녁, 본 프로젝트의 재작업 2일째 되는 날이었다.

2. 니가 진짜로 원하는게 뭐야?

React로 포팅하는 과정은 생각보다 난관이 많았다. 새로운 메시지가 등장할 때 마다 하단으로 스크롤 된다거나, 컴포넌트 별로 파일을 나누고 관리하는 익숙하지 않는 방식에 익숙해져야 한다던가, 이미지 링크의 상대 경로를 신경써야 하는 문제라던가… 대부분의 문제가 정리된 것은 오늘 늦은 저녁, 지금 글을 적고 있는 3일차가 되어서야 어느 정도 마무리가 되었다.

기능적으로는 간단한 로컬 json 파일의 데이터들을 불러와, 텍스트와 이미지의 묶음을 메시지로 구분해 표시해주는 방식으로 정리하게 된 것이다. 하지만 가장 중요한 부분은 아직 정리가 되지 않았다. 이 툴을 이용해서 도대체 진정으로 무엇을 할 것인가가 결정되지 않았다는 것이다.

애초에 하려 했던 유튜브 컨텐츠 만드는 툴로 쓰기엔 너무 많은 길을 와버렸다는 생각이 든다. 그렇다고 다른 방향으로 활용하기에는 구현하고 고려해야할 부분 투성이라는 점이 걱정거리가 된 것이다. 사실 이러한 문제와 고민들을 먼저 해결한 후에 작업을 시작했어야 했을텐데… 당분간 감기에 찌든 몸으로 골치 썩을 생각을 하니 걱정이 이만 저만이 아니다.
